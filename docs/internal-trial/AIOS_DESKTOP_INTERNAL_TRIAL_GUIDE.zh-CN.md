# AIOS Desktop 内部试用指南

本文只用于 AIOS Desktop 可信内部试用，不是公开安装或公开发行说明。

## 基线与结论

- Commit：`23e71f226cdd99ffc1973954f9d70ca0058c0474`
- 结论：GO，仅限可信内部 packaged Mac trial。
- 状态：不适合公开发布。
- 当前内部构建：unsigned，未公证，未配置 updater-ready 分发。

## 试用人员

只邀请受信任的 Mac 同事。测试目标是确认 AIOS Desktop 的本机只读元数据体验是否清楚、稳定、不会越界。

不要让试用人员扫描真实 home、整块磁盘、公司私有工作区、生产仓库、凭据目录或浏览器 profile。

## 打开内部构建

如果本机已有 P4A 产物，可从下列路径打开：

- DMG：`src-tauri/target/release/bundle/dmg/AIOS Desktop_0.1.0_aarch64.dmg`
- App bundle：`src-tauri/target/release/bundle/macos/AIOS Desktop.app`

如果这些路径不存在，不要为本次试用临时 rebuild。请向试用协调人索取匹配芯片架构的内部 unsigned Mac 产物。Apple Silicon 使用 `aarch64` 产物；Intel Mac 应使用 Intel 产物。

macOS 可能提示该构建来自未识别开发者或无法验证。这是内部 unsigned build 的预期风险提示，不代表它适合公开安装。遇到公司安全策略拦截时，停止测试并反馈，不要绕过组织安全要求。

## 首次试用流程

1. 打开 AIOS Desktop。
2. 确认主导航为：首页、技能、MCP、高级。
3. 在首页点击“开始查找”。
4. 阅读引导说明，确认应用不会自动扫描、不会上传、不会修改文件、不会读取密钥值、不会启动或调用 MCP。
5. 选择一个安全的小文件夹，例如合成 fixture 或明确的测试项目。
6. 确认选择文件夹后不会自动开始查找。
7. 进入“查找位置”或对应的手动查找区域。
8. 手动点击开始查找。
9. 观察运行中、完成或失败状态是否清楚。
10. 查找完成后查看首页摘要、技能列表、MCP 服务和 MCP 工具结果。

## 预期行为

技能详情应说明技能名称、用途、来源、可用工具、使用方式、状态和需要处理的问题。详情页不应执行技能，不应修改技能文件，也不应自动复制配置。

MCP 页面应展示本机 MCP 服务和工具的静态元数据，例如服务名、工具名、配置来源、状态、环境变量名称和人工处理建议。MCP 工具或服务信息是静态/未验证的，除非界面明确标注，否则不承诺 MCP 实时运行状态。

MCP 详情可以显示环境变量名称，但不能显示环境变量值、密钥、token、密码、Cookie 或登录会话。主机信息应收敛为 hostname-only，不应暴露敏感连接细节。

## 禁止事项

- 不扫描 home root、`/Users`、`/Volumes`、`/System`、`/Library` 或整块磁盘。
- 不扫描包含密钥、隐私内容、客户数据、浏览器 profile、凭据或生产配置的目录。
- 不启动 MCP 服务、不连接 endpoint、不调用 MCP 工具。
- 不上传查找结果、原始日志、截图或私有路径。
- 不把本内部构建转发给非试用人员。
