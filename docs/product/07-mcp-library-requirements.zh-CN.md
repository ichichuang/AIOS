# MCP 工具库需求

## 目标

MCP 工具库要让用户知道这台电脑上配置了哪些 MCP 服务、每个服务提供哪些工具、在哪里配置、能做什么、状态是否正常、哪里坏了或有风险、用户可以怎样人工处理。

MCP 页面只解释和展示，不启动 MCP 服务，不调用 MCP 工具。

## MCP 的用户解释

页面第一次出现 MCP 时必须解释：

MCP 是一种让 AI 应用连接外部工具的方式。AIOS Desktop 只显示这台电脑上已经配置的 MCP 服务和工具，不会直接运行它们。

## 服务列表

每个 MCP 服务必须显示：

- 服务名称。
- 来源工具，例如 Codex、Claude 或其他本地配置。
- 配置来源。
- 它能做什么。
- 状态。
- 工具数量。
- 是否需要处理。

服务列表顶部必须显示：

- MCP 服务总数。
- MCP 工具总数。
- 需要处理的服务数量。
- 搜索框。
- 状态筛选。
- 来源筛选。

## 工具列表

每个服务下必须显示它提供的工具。

每个工具显示：

- 工具名称。
- 简短用途。
- 来自哪个 MCP 服务。
- 是否可见。

如果无法读取工具列表，显示：

- 暂时无法读取工具列表。AIOS Desktop 不会启动服务来获取更多内容。

## 配置来源

配置来源必须用普通用户能理解的方式显示：

- 来自 Codex 配置。
- 来自 Claude 配置。
- 来自项目配置。
- 来自手动添加的位置。
- 来源不明。

高级详情可以显示路径提示、命令名称和环境变量名称。不能显示环境变量值或密钥。

## 健康状态

| 状态 | 含义 |
| --- | --- |
| 可见 | 找到了服务配置和基本信息。 |
| 可能可用 | 配置看起来完整，但没有实际启动验证。 |
| 需要处理 | 缺少依赖、路径异常、版本不固定或来源不可读。 |
| 来源不明 | 找到了线索，但无法判断配置来自哪里。 |
| 无法读取 | 配置位置无法读取或已不存在。 |

因为 AIOS Desktop 不启动 MCP 服务，所以不能承诺“已运行”或“已连接”。

## 缺少依赖提示

当发现可能缺少依赖时，应显示用户能理解的提示：

- 可能需要先安装 `{dependency}`。
- 这个服务使用 `{command}`，但 AIOS Desktop 不能确认它是否存在。
- 这个服务可能会通过 `npx` 拉取包，建议开发者确认版本。
- 这个服务需要环境变量 `{envName}`，但 AIOS Desktop 不会读取它的值。

## 用户可见修复建议

修复建议必须是人工指导，不是自动操作：

- 请确认相关工具已经安装。
- 请确认配置来源仍然存在。
- 请确认需要的环境变量已经在你的 AI 工具里配置。
- 如果看到版本不固定，请让开发者固定版本或确认来源可信。
- 如果来源不可读，请手动选择更具体的配置文件夹。
- 如果你不认识这个服务，可以先把它标记为需要处理。
- 高级用户可以打开来源位置查看。

禁止提供：

- 一键安装。
- 一键修改配置。
- 一键启动服务。
- 一键调用工具。
- 显示或复制密钥值。

## 分组

默认分组：

- 可见。
- 需要处理。
- 来源不明。

高级分组：

- Codex。
- Claude。
- 项目配置。
- 手动添加。
- 可能远程连接。
- 可能需要依赖。

高级分组不能替代普通用户默认分组。
