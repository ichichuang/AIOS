# 产品验收标准

## 文档验收

- 产品文档位于 `docs/product/`。
- **必须包含并优先引用 `docs/product/00-aios-desktop-product-definition.zh-CN.md` 作为最高权威产品定义。**
- 至少包含产品定位、用户需求、首次使用流程、信息架构、术语指南、技能库需求、MCP 工具库需求、隐私与安全需求、产品验收标准。
- 文档使用简体中文。
- 文档默认面向普通用户。
- 文档明确说明本阶段不做实现，不改代码。
- 文档明确说明 Scripts、Reports、Policies、Validators、Legacy、原始查找诊断和内部调试概念不是一级产品模块。

## 界面简单性验收

- 主导航只有：首页、技能、MCP、高级。
- 首页必须 visibly 分离以下模块：Hero、本地整理内容概览、项目能力状态、技能入口、MCP 入口、本地隐私说明。P6C 阶段首页不显示“需要处理”模块。
- 项目能力概览必须诚实说明项目级 AI 能力组织在 P6C 阶段暂不可用；不得使用 `resourceCorpus.projectMap`、路径或推断名称渲染项目。
- 首屏能看到已整理的本地技能数量、已记录的 MCP 服务数量和工具名称线索数量；P6C 阶段不显示需要处理的数量。
- 普通用户不需要理解内部工程词汇也能完成首次查找。
- 高级内容不会出现在首页默认区域。
- 任意页面都不能把脚本、报告、旧入口、检查器或内部诊断作为一级模块展示。

## 技能数量准确性验收

- 技能默认浏览按任务和功能分类。
- 技能来源只作为筛选、详情来源说明和高级来源视图。
- 首页技能总数必须等于技能库当前可见来源的去重结果。
- 重复技能不能重复计入“可用技能”。
- 来源不明项目不能计入“可用技能”，除非已经确认是完整技能。
- 已损坏技能可以计入“需要处理”，但不能计入“可用技能”。
- 技能详情必须能说明该技能来自哪里。
- 技能详情必须能说明哪个 AI 工具可以使用它。
- 技能详情必须提供使用方式或说明为什么暂时无法判断。

## MCP 可见性验收

- MCP 默认浏览按服务展示。
- MCP 状态和来源只作为筛选、详情状态和高级来源视图。
- 首页显示 MCP 服务数和 MCP 工具数。
- MCP 页面显示服务列表和每个服务的工具列表。
- MCP 页面显示配置来源、用途、问题和人工处理建议。
- 无法读取工具列表时，页面必须说明不会启动服务来获取更多内容。
- 页面不能显示密钥值、令牌值或环境变量值。
- 页面不能提供启动服务或调用工具的操作。

## 普通用户一键流程验收

- 首次启动不自动查找。
- 首页有明确的“开始查找”按钮。
- 点击开始前说明会查找什么、不会做什么。
- 查找过程中显示简单进度。
- 查找结束后显示技能数量、MCP 数量和需要处理的数量。
- 没有结果时，引导用户手动选择文件夹。

## 高级开发者来源分组验收

- 高级页可以按来源查看技能和 MCP。
- 来源至少支持全局来源、项目来源、Codex、Claude、手动选择的文件夹、来源不明。
- 高级页可以显示路径提示，但必须隐藏敏感部分。
- 高级页可以显示跳过项和问题摘要，但不能显示原始日志或密钥。
- 高级页不能提供自动修改配置、安装依赖、执行脚本或启动 MCP 的操作。

## 术语合规验收

- 普通用户文案使用“本机”“已整理”“已记录”“项目”“技能”“MCP 服务”“能做什么”“适合什么时候使用”“可用于”“来源”“来自本地配置记录”“未进行实时连接验证”“工具名称线索”“暂未整理项目级 AI 能力”“暂无可整理的内容”等简单词。
- 禁用“资源语料”“corpus”“scanner”“资源检查器”“Token 压力”“元数据质量”“数据库记录”“原始资源”“runtime”“schema”“command contract”“raw ID”“absolute path”“internal status”等工程词汇出现在普通用户主界面。
- 不得把本地聚合数据描述为全电脑或全局范围。
- 不得把 MCP `sourceLabel` 描述为应用兼容性。
- 不得把 MCP 工具名称线索描述为已验证工具。
- MCP 第一次出现时有中文解释。
- 技能第一次出现时有中文解释。
- 禁用术语只允许出现在术语指南的禁用表、信息架构的移除说明或开发者补充中。
- 任何面向普通用户的标题、按钮、空状态、错误文案，都不能使用难懂工程词汇。

## 隐私与安全验收

- 文档明确说明结果只保存在本机。
- 文档明确说明不上传查找结果。
- 文档明确说明不读取密钥、令牌、密码、Cookie、登录会话或环境变量值。
- 文档明确说明不执行脚本、不启动 MCP 服务、不调用 MCP 工具。
- 文档明确说明清空本地记录不会删除用户文件。

## 本阶段停止线

完成这些文档后停止。不得开始 UI、扫描、存储、构建、工作流或发行实现。

## P6D-D 技能页面、行与详情检查器验收

### 可理解的作用域

- 用户首次进入 Skills 页面即可通过 Tab 区分：全部技能、全局技能、项目技能、范围未整理。
- 每个作用域 Tab 显示真实计数；当前作用域视觉突出。
- 项目作用域下显示项目选择器，首项为“全部项目”；相同 projectLabel 但不同 projectId 的项目作为独立选项。
- 项目身份仅来自显式 `scopeSummary.projects`，不得从路径、文件夹、来源应用或 `source_kind` 推断。
- Mixed Skill 同时出现在全局和项目视图，范围未整理视图只包含 `classification === "unknown"` 的 Skill。

### 能力作为首要浏览维度

- 能力导航在视觉上必须比来源/状态更突出。
- 默认按能力分组显示结果；选择单个能力时显示单一结果列表。
- 禁止以提供者名称、来源应用或 source_kind 作为分类标题或默认分组。
- 来源和状态仅作为“更多筛选”中的次要控制。

### 行级可读性

- 每行以技能显示名称为首，其次是范围标签和必要状态。
- 普通 available 状态不重复显示“可用”chip。
- 用途说明直接显示；使用场景以“适合：”为前缀。
- 兼容工具最多显示两个标签，剩余以 +N 收起；Unknown 工具不显示。
- 已知来源作为次要元数据；不重复显示“来源不明”chip。
- 原始技术名、路径、ID、重复计数、扫描器信息不得作为行主要内容。
- 用途和使用场景同时缺失时，只显示一次“暂未记录详细用途”。

### 详情检查器验收

- 主标题区显示能力分类、显示名称、范围摘要；原始技术名不进入主标题区。
- 主体区顺序固定为：它能做什么、适合什么时候用、如何使用、可在哪些 AI 工具中使用、归属范围。
- 工具区只展示显式 `availableInTools`，为空时隐藏。
- 多个主要字段缺失时，只显示一次“这个技能的详细说明还不完整，AIOS 只展示当前已记录的信息。”
- 来源与记录、检查信息、高级信息默认折叠。
- 高级信息中包含原始技术名、别名、标签、能力、来源数、安全边界。
- 私有绝对路径、原始 ID、扫描器内部状态不得作为普通内容展示。

### 响应式与动效

- 结果面板和检查器各自独立滚动；Shell 与导航保持固定。
- 900px 检查器打开状态下，主内容不得塌成 rail 宽度列；沿用 P5E-C 保护。
- 最窄桌面宽度下，能力 rail 退化为紧凑 Select；无水平文档滚动、无重复滚动条。
- 静态行、卡片、容器禁止 transform / translate / scale / lift 悬停动效；仅允许 background、border、opacity 过渡。

### 验证与质量

- 所有 SkillsModule、SkillRow、SkillDetailInspector 渲染测试通过。
- 使用真实应用外壳与临时合成 Tauri IPC 桥接完成至少 12 个状态的视觉 QA。
- 视觉 QA 必须覆盖：1440px 全部/全局/项目+检查器、1280px 分组结果、1024px 选中能力+检查器、900px 检查器打开、最窄宽度紧凑选择器、空状态、搜索无结果。
- 临时 QA 脚本和截图仅存放于 `/tmp/aios-p6d-d-vqa/`，不得留在仓库中。
- 不得添加假数据到生产代码；合成数据仅用于测试和临时视觉 QA。
