# 用户需求

## 默认用户：普通用户

普通用户需要一个简单答案：我的电脑上到底有什么 AI 能力可以用。

必须满足：

- 一眼看到 AI 技能总数和 MCP 工具总数。
- 看懂每个技能能做什么，不需要理解文件结构。
- 看懂技能来自哪里，例如“Codex 插件”“本地共享技能”“项目文件夹”。
- 看懂哪个 AI 工具可以使用这个技能，例如 Codex、Claude 或 Agents。
- 看懂怎样使用技能，例如“在 Codex 里输入 `$skill-name`”或“复制这段提示给 AI”。
- 可以按来源、状态和问题整理技能。
- 看懂哪些项目需要处理，例如“找不到文件”“来源重复”“缺少说明”“可能需要安装依赖”。
- 第一次打开时不会被大量工程词汇、历史模块或调试信息淹没。

## 第二类用户：高级开发者

高级开发者需要更准确的来源和排查信息。

重要需求：

- 查看技能来自哪个工具、哪个文件夹、哪个插件或哪个项目。
- 按来源分组，例如全局来源、项目来源、Codex、Claude、手动选择的文件夹。
- 查看 MCP 服务的配置来源、启动命令名称、环境变量名称、工具列表和健康状态。
- 看到重复技能、断开的来源、版本不固定、缺少依赖等问题。
- 可以在“高级”里添加或管理要查找的来源。
- 可以导出或复制问题摘要，方便后续人工处理。

## 优先级

### 必须

- 首页显示技能数量、MCP 数量和需要处理的数量。
- 技能库可以搜索、筛选、分组和查看详情。
- MCP 页面可以查看服务、工具、来源和状态。
- 首次使用有一个清晰的一键查找流程。
- 所有普通用户文案使用自然中文。
- 隐私边界清楚：本地查看，不上传，不读取敏感内容。

### 重要

- 高级来源设置可以让开发者添加特定文件夹。
- 技能详情能给出可靠的使用说明。
- 技能详情能说明哪个 AI 工具可以使用它。
- MCP 详情能说明缺少依赖时用户可以做什么。
- 重复、损坏、来源不明的项目有统一状态和解释。

### 以后再做

- 自动修复。
- 安装缺失依赖。
- 修改技能或 MCP 配置。
- 运行 MCP 工具。
- 团队共享、云端同步或远程备份。

## 非目标

- 不把 Scripts、Reports、Policies、Validators、Legacy 做成一级模块。
- 不把原始查找日志、内部错误栈、调试字段放到普通用户界面。
- 不要求普通用户理解数据库、运行时、内部索引或工程扫描细节。
- 不默认查找整台电脑。
- 不自动修改任何本机文件或 AI 工具配置。
