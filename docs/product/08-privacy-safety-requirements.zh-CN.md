# 隐私与安全需求

## 基本承诺

AIOS Desktop 帮用户查看本机 AI 技能和 MCP 工具的基本信息。它不上传结果，不执行脚本，不启动 MCP 服务，不修改用户文件。

## 应用会读取什么

AIOS Desktop 可以读取：

- 技能名称。
- 技能说明。
- 技能所在来源。
- MCP 服务名称。
- MCP 工具名称。
- 配置文件里非敏感的字段名称。
- 文件夹和文件的基本信息，例如名称、大小、修改时间。
- 查找过程中产生的问题摘要。

这些内容只用于帮助用户理解本机 AI 技能和 MCP 工具。

## 应用永远不读取或展示什么

AIOS Desktop 不读取、不保存、不展示：

- 密钥。
- 令牌。
- 密码。
- 私钥。
- 浏览器 Cookie。
- 登录会话。
- 环境变量的值。
- 聊天记录。
- 邮件内容。
- 私人文档正文。
- 文件完整内容，除非未来另有明确的用户确认设计。

## 应用不会做什么

AIOS Desktop 不会：

- 自动查找。
- 默认查找整台电脑。
- 上传查找结果。
- 删除用户文件。
- 修改技能文件。
- 修改 MCP 配置。
- 执行脚本。
- 启动 MCP 服务。
- 调用 MCP 工具。
- 修改系统设置。
- 绕过系统权限。

## 什么留在本机

以下内容留在用户电脑上：

- 查找结果。
- 用户添加的查找位置。
- 最近一次查找时间。
- 问题摘要。
- 本地应用设置。

清空本地记录只删除 AIOS Desktop 自己保存的结果，不删除用户文件或外部工具配置。

## 用户必须确认的动作

必须由用户明确点击或确认：

- 第一次开始查找。
- 手动选择文件夹。
- 添加新的查找位置。
- 清空本地记录。
- 进入高级全电脑查找。

高级全电脑查找必须放在“高级”里，并且默认关闭。普通用户不应该在首次使用中看到它作为推荐路径。

## 错误和跳过

如果某个位置无法读取：

- 应显示“已跳过”，不要把它说成严重错误。
- 应告诉用户可以选择更具体的文件夹。
- 不要求用户降低系统安全设置。

如果发现疑似敏感内容：

- 不显示原始值。
- 只显示“已隐藏敏感内容”。
- 把项目标记为需要处理。

## 开发者补充

“基本信息”指文件名、文件夹提示、大小、修改时间、配置项名称、服务名称、工具名称和状态摘要。它不包含密钥值、正文内容或登录材料。
