# 用户旅程与首次使用流程

## 旅程目标

用户第一次打开 AIOS Desktop 时，应先理解“这个应用会帮我找本机 AI 技能和 MCP 工具”，然后通过一个明确按钮开始查找。用户不需要先学习工程概念。

## 首次启动

首次启动时显示首页空状态：

- 标题：还没有查找这台电脑上的 AI 技能
- 说明：点击开始后，AIOS Desktop 会在本机查找 AI 技能和 MCP 工具的基本信息。
- 主按钮：开始查找
- 次按钮：手动选择文件夹
- 隐私提示：查找结果只保存在这台电脑上，不上传到云端。

首次启动不能自动查找。必须由用户点击开始。

## 一键查找流程

一键查找是普通用户默认路径。

流程：

1. 用户点击“开始查找”。
2. 应用显示将要做什么：查找 AI 技能、MCP 工具和需要处理的问题。
3. 应用显示不会做什么：不读取密钥，不上传结果，不执行脚本，不启动 MCP 服务。
4. 用户确认后开始。
5. 过程中显示简单进度：正在查找技能、正在查找 MCP、正在整理结果。
6. 结束后进入结果概览。

一键查找的结果页面必须避免内部细节。普通用户只需要看到数量、分类和问题摘要。

## 结果概览

首页结果概览显示：

- AI 技能数量。
- MCP 服务数量。
- MCP 工具数量。
- 需要处理的项目数量。
- 最近一次查找时间。
- 主要来源，例如全局来源、项目来源、Codex、Claude、手动选择的文件夹。

当没有找到内容时：

- 标题：没有找到 AI 技能或 MCP 工具
- 说明：你可以手动选择一个包含技能或 MCP 配置的文件夹。
- 操作：选择文件夹

## 技能详情流程

用户从技能列表进入详情后，应该看到：

- 技能名称。
- 一句话用途。
- 适合什么时候使用。
- 使用方法。
- 哪个 AI 工具可以使用它。
- 来源。
- 状态。
- 需要处理的问题。

如果技能可用，主操作是复制使用方式或打开来源位置说明。不能提供会修改文件或执行脚本的操作。

## MCP 详情流程

用户从 MCP 页面进入详情后，应该看到：

- MCP 服务名称。
- 它提供哪些工具。
- 它来自哪个配置位置。
- 当前状态。
- 可能缺少什么依赖。
- 用户可以怎样确认或修复。

页面不能启动 MCP 服务，不能调用 MCP 工具，不能显示密钥值。

## 高级来源设置流程

高级用户可以进入“高级”添加来源。

流程：

1. 用户进入高级。
2. 选择“添加查找位置”。
3. 选择一个明确的文件夹。
4. 给来源命名，例如“公司项目”“本地共享技能”。
5. 选择用途：技能、MCP、两者都查找。
6. 点击“查找这个位置”。
7. 结果回到首页、技能和 MCP 页面展示。

高级流程应提醒用户：选择越大的文件夹，查找越慢，也更容易出现跳过项。建议选择具体项目或工具文件夹。

高级来源分组至少支持：全局来源、项目来源、Codex、Claude、手动选择的文件夹。
