# AIOS Desktop 产品定义

## 状态

权威产品定义。本文档是 AIOS Desktop 当前产品方向的唯一最高权威来源。界面、文案、测试、验收和其他产品文档均须以此为准。

## 产品身份

AIOS Desktop 是一款**本地优先的桌面产品**，用于整理 AIOS 在当前电脑上已经可获得的 AI 技能与 MCP 服务配置记录。它帮助普通用户理解已记录了什么、每个能力能做什么、适合什么时候使用，以及这些记录来自哪里。

它不是以下任何一种产品：

- 不是全磁盘扫描器或安全扫描器。
- 不是系统管理控制台。
- 不是资源治理控制台。
- 不是数据库浏览器或元数据质量工作台。
- 不是工作流引擎或脚本执行器。
- 不是 MCP 运行时管理器或 MCP 服务启动器。
- 不是开发者诊断仪表板。
- 不是云同步服务或团队协作平台。

## 目标用户

**默认用户是普通用户**：他们可能在日常使用 AI 工具，但不一定理解技能文件、MCP 配置、命令行或开发环境细节。普通用户打开应用后，应能在不学习工程概念的情况下回答“我的电脑上有哪些 AI 能力可用”。

**第二类用户是高级开发者**：他们需要来源、路径、配置位置、重复来源和问题摘要。这些能力必须收敛在“高级”模块或明确标注为开发者诊断的区域，不能成为普通用户的默认体验。

## 用户要回答的问题

AIOS Desktop 的界面和信息架构必须让用户容易回答：

1. 这台电脑上有哪些 AI 能力可用？
2. 有哪些技能可用？
3. 每个技能能做什么？
4. 适合什么时候使用这个技能？
5. 哪些 AI 应用或智能体可以使用这个技能？
6. 这个技能来自哪里？
7. 有哪些 MCP 服务已记录？
8. 每个 MCP 服务提供什么能力？
9. 哪些 AI 应用可以使用这个 MCP 服务？
10. MCP 信息是来自本地配置记录，还是已经过实时验证？

项目级能力区分和“需要处理”的可行动项在 P6C 阶段暂不支持；它们属于后续迭代，当前界面必须诚实说明不可用。

## 核心产品承诺

- **本地优先**：所有查找结果只保存在用户自己的电脑上。
- **只读展示**：不上传、不修改用户文件、不执行脚本、不启动 MCP 服务、不调用 MCP 工具。
- **普通用户优先**：默认界面使用自然中文，避免工程术语。
- **诚实边界**：不伪造项目、不伪造技能、不伪造 MCP 服务、不伪造实时运行状态。
- **安全第一**：不读取密钥、令牌、密码、Cookie、登录会话或环境变量的值。

## 产品边界

AIOS Desktop 只帮助用户理解和查看。以下动作不属于当前产品承诺：

- 修改本机配置或删除文件。
- 安装依赖或修复环境。
- 执行脚本、启动服务或连接远程端点。
- 自动扫描整台电脑或系统目录。
- 将结果同步到云端或共享给团队。

如果未来要加入修复、安装、同步等能力，必须另起独立需求与确认流程，不能混入当前产品定义。

## 安全承诺

- 不自动查找；首次启动不扫描任何目录。
- 不上传查找结果。
- 不读取密钥、令牌、密码、浏览器 Cookie、登录会话或环境变量的值。
- 不执行脚本，不启动 MCP 服务，不调用 MCP 工具。
- 不修改技能文件、MCP 配置、系统设置或用户文件。
- 清空本地记录只删除 AIOS Desktop 自己保存的结果，不删除用户文件。
- 高级全盘查找必须放在“高级”里，并需要用户明确确认。

## 一级导航

一级导航**只保留四项**：

| 导航 | 用途 |
| --- | --- |
| 首页 | 查看本机 AI 能力概览、项目能力状态、技能入口、MCP 入口和一键查找入口。 |
| 技能 | 按任务和功能分类浏览技能，查看每个技能的用途、使用方法和来源。 |
| MCP | 按服务浏览 MCP 配置记录，查看每个服务提供的工具名称线索和来源。 |
| 高级 | 给需要排查的人查看查找位置、来源分组、问题摘要和本地记录控制。 |

禁止将 电脑、项目、资源、扫描、报告、脚本、策略、验证器、旧入口 等作为一级导航。

电脑范围和项目范围必须在 首页、技能、MCP 内部作为清晰模块、分区或范围控制呈现，不能扩展为多级管理控制台。

## 产品模块

### 1. 电脑概览（Computer overview）

目的：

- 汇总 AIOS Desktop 已经整理的本机 AI 能力记录。
- 显示有用总数和清晰入口。
- 解释结果来自 AIOS Desktop 自己保存的本地基本信息。
- 绝不暗示全电脑发现、全电脑范围或实时验证。

必须展示：

- 已整理的 AI 技能数量。
- 已记录的 MCP 服务数量。
- 可选的 MCP 工具名称线索数量（使用不确定措辞）。
- 最近查找时间。
- 一键查找入口。

P6C 阶段不展示：

- 需要处理的项目数量（`needsAttentionCount` 不是安全的普通用户可行动项计数）。

### 2. 项目概览（Project overview）

目的：

- 诚实说明项目级 AI 能力组织在 P6C 阶段暂不可用。
- 绝不编造项目、从不支持的路径推断项目所有权，也不使用 `resourceCorpus.projectMap` 渲染项目列表。
- 当项目级元数据模型未来可用时，再展示项目特定的技能或 MCP 配置。

行为：

- 首页项目能力分区标题为“项目能力”。
- 主文案显示“暂未整理项目级 AI 能力”。
- 辅助文案说明“AIOS 当前还不能可靠地区分项目专属的技能和 MCP 服务”。
- 不列出项目行、项目计数、路径或推断的项目名称。
- 不提供操作按钮或跳转到“高级”的链接。

### 3. 技能（Skills）

目的：

- 按用户任务、功能或类别优先浏览。
- 显示技能名称、它能做什么、适合什么时候使用、类别或任务、范围、兼容的 AI 应用或智能体、来源或出处。
- 将来源和提供者作为次级元数据和筛选条件。
- 在现有元数据支持时，清晰区分电脑-wide 和项目-specific 技能。

行为：

- 默认按任务/功能分类，不按来源分类。
- 每个可见技能行或卡片优先展示：技能名称、 plain-language 用途、适合什么时候使用、类别、范围、兼容 AI 应用、来源。
- 不重复显示未知或不可用的标签。
- 详情视图默认简洁可扫描；路径、manifest、Token 估算等归入高级来源信息。

### 4. MCP

目的：

- 按 MCP 服务优先浏览本地配置记录。
- 解释每个服务提供什么能力。
- 将工具名称线索作为从属于服务的信息；这些线索是未经验证的工具名称提示，不是已验证的工具列表。
- 显示兼容的 AI 应用（当已知时）。
- 将状态、来源、范围作为次级元数据或筛选条件。
- 清晰区分本地配置记录与实时连接验证。

行为：

- 默认按服务展示；不把单个工具作为同级一级模块。
- 每个可见服务行或卡片优先展示：服务名称、plain-language 用途、提供的能力、范围、兼容 AI 应用、静态配置状态、来源。
- `sourceLabel` 代表配置来源（provenance），不代表应用兼容性（compatibility）。
- 使用诚实措辞，如“来自本地配置记录”“未进行实时连接验证”“工具名称线索”。
- 不在普通界面使用绿色“已连接/已运行”等暗示实时验证成功的语言，除非数据确实来自已验证的实时状态。
- 不提供连接、启动、停止、测试或调用工具的操作。

### 5. 需要处理（Attention）

目的：

- 只展示需要用户注意的可行动项目。
- 避免用技术性诊断吓到用户。
- 解释问题、影响和下一步安全动作。
- 不把缺失的可选元数据当作严重错误。

行为：

- P6C 阶段不在首页显示“需要处理”总数或结论；`needsAttentionCount` 不是安全的普通用户可行动项计数。
- 未来可在技能和 MCP 页面支持按“需要处理”筛选。
- 使用普通用户能理解的状态：需要处理、重复、已损坏、来源不明。
- 当前技术诊断保留在“高级”模块。

### 6. 高级（Advanced）

目的：

- 承载诊断、支持详情、扫描器元数据、原始出处、报告和历史入口。
- 防止技术概念泄漏到普通主要流程。

行为：

- 将查找位置、来源分组、跳过项摘要、错误摘要、本地数据控制、旧入口示例等收在此模块。
- 所有入口文案必须标注“高级支持”或类似提示。
- 高级模块不能成为默认首页，也不能主导视觉层级。

## 术语规则

普通用户界面优先使用自然中文。推荐：

- 本机、当前电脑
- 已整理、已记录
- 项目
- 技能
- MCP 服务
- 能做什么
- 适合什么时候使用
- 可用于
- 来源
- 来自本地配置记录
- 未进行实时连接验证
- 工具名称线索
- 暂未整理项目级 AI 能力
- 暂无可整理的内容

普通用户界面避免：

- 资源语料 / corpus
- scanner / 资源检查器
- Token 压力 / 元数据质量
- 数据库记录 / 原始资源
- runtime / schema / command contract
- raw ID / absolute path / internal status
- 技术 severity codes
- 把本地聚合描述为“全电脑”“全局”或“整台电脑发现”
- 把 MCP `sourceLabel` 称为“兼容性”
- 把 MCP 工具名称线索称为“已验证工具”

允许在“高级”或开发者诊断区域使用工程术语，但不得作为普通用户主功能名称或说明。

## 空状态规则

- 首页空状态：标题“还没有查找这台电脑上的 AI 技能”；说明“点击开始后，AIOS Desktop 会查找本机 AI 技能和 MCP 工具的基本信息。”；主按钮“开始查找”；次按钮“手动选择文件夹”。
- 技能空状态：标题“还没有找到 AI 技能”；说明“开始查找后，这里会显示技能名称、用途、来源和使用方法。”
- MCP 空状态：标题“还没有找到 MCP 服务”；说明“开始查找后，这里会显示本机已配置的 MCP 服务和工具。”
- 项目概览空状态：标题“暂未整理项目级 AI 能力”；说明“AIOS 当前还不能可靠地区分项目专属的技能和 MCP 服务。”
- 搜索无结果：标题“没有匹配结果”；说明“换一个关键词，或清除筛选后再试。”

所有空状态必须诚实，不伪造数据，不把“未扫描”渲染为错误。

## 静态与未验证元数据措辞

- 必须明确说明 AIOS Desktop 只读取本地已保存的基本信息。
- 必须说明不启动 MCP 服务、不连接端点、不调用工具。
- 对 MCP 服务状态使用“来自本地配置记录”“未进行实时连接验证”“可能可用”等措辞。
- 不得在缺乏实时验证的情况下使用“已连接”“已运行”“已验证可用”等语言。

## 显式非目标

- 不把 AIOS Desktop 重新定义为通用资源治理平台。
- 不恢复或重建旧的一等模块（Scripts、Reports、Project Packs、Policies、Validators、Legacy）作为默认导航。
- 不将技术来源分组（registry、runtime view、active entrypoint、distilled、archived）作为普通用户默认浏览骨架。
- 不将状态、风险、来源提升到与任务/功能同级的默认展示位置。
- 不通过扫描器扩大读取范围来补全产品数据。

## 产品验收原则

- 产品文档、界面文案、测试断言使用同一产品定义。
- 一级导航严格为 首页 / 技能 / MCP / 高级。
- 首页 visibly 分离 Hero、本地整理内容概览、项目能力状态、技能入口、MCP 入口、本地隐私说明。
- P6C 阶段首页不显示“需要处理”或基于 `needsAttentionCount` 的结论。
- 本地聚合数据使用“已整理”“已记录”等措辞，不描述为全电脑或全局范围。
- 项目能力诚实说明暂不可用，不使用 `resourceCorpus.projectMap`、路径或推断名称渲染项目。
- 技能默认 category/task/function-first；来源 secondary。
- MCP 默认 service-first；工具 subordinate；工具名称线索不称为已验证工具；`sourceLabel` 不称为兼容性。
- 静态 MCP 元数据不声称实时验证。
- 无伪造项目、技能、MCP 服务或实时状态。
- 普通用户界面不出现禁用工程术语。
- 高级模块承载诊断概念，但不主导默认体验。
