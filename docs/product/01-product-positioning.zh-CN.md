# 产品定位

## 一句话

AIOS Desktop 是一款本地桌面应用，帮助用户查看、理解、整理和使用自己电脑上的 AI 技能和 MCP 工具。

## 产品承诺

用户打开 AIOS Desktop 后，应该能很快回答这些问题：

- 我这台电脑上一共有多少个 AI 技能？
- 每个技能能帮我做什么？
- 这个技能来自哪里？
- 哪个 AI 工具可以使用它？
- 我该怎样使用它？
- 我有哪些 MCP 工具？
- 哪些技能或 MCP 工具需要我注意？

## 它是什么

- 本地查看工具：主要读取这台电脑上的技能和 MCP 基本信息。
- 技能说明书：把技能的用途、来源、使用方式和状态讲清楚。
- MCP 工具清单：列出已配置的 MCP 服务和它们提供的工具。
- 本地整理助手：帮助用户按来源、状态和问题整理技能与 MCP。
- 问题提醒器：把缺失、重复、来源不明、可能无法使用的项目清楚标出来。
- 普通用户优先的桌面应用：默认界面必须简单，不能像工程后台。

## 它不是什么

- 不是复杂的 AI 资源管控后台。
- 不是全电脑文件浏览器。
- 不是自动修复工具。
- 不是脚本执行器。
- 不是 MCP 运行器，不能在详情页里直接启动 MCP 服务或调用 MCP 工具。
- 不是云同步服务，不把本机结果上传到云端。
- 不是面向内部工程概念的调试面板。

## 服务对象

默认用户是普通用户。他们可能知道自己在使用 AI 工具，但不一定知道技能文件、MCP 配置、命令行或开发环境细节。

高级开发者是第二类用户。他们需要查看来源、路径、配置位置、缺失依赖、重复来源和更细的问题说明，但这些内容必须收在“高级”里，不能成为普通用户的默认体验。

## 产品边界

AIOS Desktop 只帮助用户理解和查看。任何修改本机配置、删除文件、安装依赖、执行脚本、启动服务、连接远程服务的动作，都不属于当前产品承诺。

如果未来要加入修复能力，必须另起需求和确认流程，不能混入当前产品定义。
