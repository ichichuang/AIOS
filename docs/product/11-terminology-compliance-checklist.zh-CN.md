# 术语合规检查清单

状态：产品文案检查标准。

本清单用于检查 AIOS Desktop 文档和界面文案是否符合新的产品方向。

## 产品定义

- [ ] 是否把 AIOS Desktop 定义为本地桌面应用？
- [ ] 是否说明它帮助用户查看、理解、整理和使用本机 AI 技能与 MCP 工具？
- [ ] 是否避免把它定义为复杂的 AI 资源管控后台？
- [ ] 是否明确普通用户是默认用户？
- [ ] 是否把高级开发者信息放到“高级”或内部文档中？

## 主导航

- [ ] 主导航是否只有：首页、技能、MCP、高级？
- [ ] 首页是否回答技能数量、MCP 数量和需要处理数量？
- [ ] 技能页是否回答用途、来源、可用工具、使用方式和问题？
- [ ] MCP 页是否回答服务、工具、配置来源、用途、问题和人工处理建议？
- [ ] 高级页是否只承载来源分组、手动文件夹、问题摘要和本地数据控制？

## 技能文案

- [ ] 是否说明技能是什么？
- [ ] 是否说明技能能做什么？
- [ ] 是否说明技能从哪里来？
- [ ] 是否说明哪个 AI 工具可以使用它？
- [ ] 是否说明怎么使用？
- [ ] 是否说明是否需要处理？

## MCP 文案

- [ ] MCP 第一次出现时是否有中文解释？
- [ ] 是否说明 MCP 服务和工具来自哪里？
- [ ] 是否说明 MCP 工具能做什么？
- [ ] 是否说明配置来源？
- [ ] 是否说明缺少依赖、来源不可读、版本不固定等问题？
- [ ] 是否只提供人工处理建议，不提供自动启动或自动修改？

## 禁用说法

普通用户文案不得使用这些说法作为主功能名或说明：

- resource corpus。
- metadata-only corpus。
- governance。
- validators。
- policies。
- scan scope。
- full-disk discovery。
- SQLite state。
- runtime view。
- registry。
- project pack。
- capability domain。
- raw scan diagnostics。
- internal scanner/debug concepts。

中文用户文案也不得把以下内容作为主模块：

- 脚本。
- 报告。
- 策略。
- 验证器。
- 旧入口。
- 项目包。
- 原始查找诊断。
- 内部调试信息。

允许例外：

- 术语指南中的禁用表。
- 文档地图、审计清单和修正报告中的说明。
- 已标明开发者内部、内部发行、内部验收或历史文档的工程内容。

## 推荐说法

- 用“查找”替代 scan。
- 用“来源”替代 registry、scope、corpus 等工程词。
- 用“基本信息”替代 metadata。
- 用“需要处理”替代风险泛化表达。
- 用“应用自己的本地记录”替代数据库实现细节。
- 用“高级全电脑查找”替代 full-disk discovery，并且只放在高级。
- 用“检查结果”替代 validators。
- 用“安全说明”替代 policies。

## 隐私与安全

- [ ] 是否说明结果只保存在本机？
- [ ] 是否说明不上传结果？
- [ ] 是否说明不读取密钥、令牌、密码、Cookie、登录会话或环境变量值？
- [ ] 是否说明不执行脚本？
- [ ] 是否说明不启动 MCP 服务、不调用 MCP 工具？
- [ ] 是否说明清空本地记录不会删除用户文件？
