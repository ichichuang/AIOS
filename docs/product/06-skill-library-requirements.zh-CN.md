# 技能库需求

## 目标

技能库要让用户看懂自己有哪些 AI 技能、每个技能做什么、从哪里来、哪个 AI 工具可以使用、怎么使用、哪些需要处理。

普通用户默认看到简单列表和清楚解释。高级信息只在展开详情或高级区域出现。

## 列表视图

每个技能卡片必须显示：

- 技能名称。
- 一句话用途。
- 来源。
- 可用工具，例如 Codex、Claude 或 Agents。
- 状态。
- 使用入口，例如 `$skill-name`。
- 是否需要处理。

列表顶部必须显示：

- 技能总数。
- 需要处理的数量。
- 搜索框。
- 来源筛选。
- 状态筛选。

## 详情视图

技能详情必须显示：

- 名称。
- 原始名称。
- 它能做什么。
- 适合什么时候用。
- 哪个 AI 工具可以使用它。
- 如何使用。
- 来源。
- 状态。
- 相关问题。
- 高级来源信息。

详情页不能执行技能，不能修改技能文件，不能自动复制到其他工具。

## 分组

默认浏览按任务和功能分类，例如写作、代码、设计、排查、资料整理和其他用途。

来源只作为筛选、详情来源说明和高级来源视图，不能替代默认的功能分类骨架。

高级用户可以在高级区域切换为更细的来源视图，例如 Codex、Claude、Agents、本地共享、项目里的技能、全局来源、手动选择的文件夹、插件带来的技能和来源不明。普通用户默认不需要看到完整路径。

## 搜索

搜索应覆盖：

- 技能名称。
- 原始名称。
- 用途说明。
- 来源名称。
- 可用工具。
- 使用入口。

搜索不执行任何命令。

## 筛选

必须支持：

- 全部。
- 可用。
- 需要处理。
- 重复。
- 已损坏。
- 来源不明。

可选支持：

- 按来源筛选。
- 按工具筛选，例如 Codex、Claude、Agents。

## 状态标签

| 状态 | 含义 |
| --- | --- |
| 可用 | 找到了说明和来源，可以使用。 |
| 需要处理 | 有问题需要用户查看。 |
| 重复 | 发现多个相同或近似的技能。 |
| 已损坏 | 来源缺文件、格式异常或无法读取。 |
| 来源不明 | 找到了线索，但还不能判断来自哪里。 |
| 未检查 | 还没有完成查找或来源未被打开。 |

## 来源标签

来源标签必须用普通用户能理解的词：

- Codex。
- Claude。
- Agents。
- 全局来源。
- 项目来源。
- 本地共享。
- 项目文件夹。
- 插件。
- 手动添加。
- 来源不明。

高级详情可以显示路径提示，但应隐藏敏感部分。

## 使用说明

每个可用技能都必须提供至少一种使用方式：

- 在 Codex 中输入的技能名。
- 在 Claude、Agents 或其他工具中的调用方式。
- 哪个工具可以使用它。
- 可复制的简短提示。
- 什么时候适合用。
- 不适合什么时候用。

如果无法判断使用方式，显示：

- 暂时无法判断使用方法。请在高级信息里查看来源。

## 异常处理

重复技能：

- 默认只展示推荐项。
- 在详情里说明还找到了哪些重复来源。
- 不自动删除或合并。

已损坏技能：

- 显示缺少什么，例如“找不到说明文件”。
- 提供人工检查建议。
- 不自动修复。

来源不明：

- 放到“来源不明”分组。
- 说明 AIOS Desktop 还不能判断它是不是完整技能。
- 不把它计入“可用技能”。

无法读取：

- 显示为“需要处理”。
- 说明可能是权限问题或文件已移动。
- 不要求用户修改系统权限。
