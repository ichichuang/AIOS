# AIOS Desktop 设计系统

文档类型：产品规划支持文档。

权威产品定义见 `docs/product/`。本设计系统服务新的主体验：首页、技能、MCP、高级。旧的脚本、报告、策略、检查器、旧示例和原始诊断不能作为普通用户主导航。

## 设计原则

- 普通用户优先。
- 信息先回答“有什么、做什么、从哪里来、怎么用、哪里需要处理”。
- 技术细节放到高级区域。
- 不做营销页，不做复杂后台。
- 不用难懂工程词汇做标题、按钮或空状态。

## 字体

- 全局使用系统无衬线字体：`ui-sans-serif`、Apple 系统字体、`Segoe UI`、`PingFang SC`、`Hiragino Sans GB`、`Microsoft YaHei`、本地可用的 `Noto Sans SC`。
- 不加载远程字体。
- 中文正文保持 14-16px，行高 1.45-1.65。
- 标题只用于层级和定位，不使用过大的营销式字号。
- 技术名、路径、命令和包名使用等宽字体，并允许自动换行。

## 色彩角色

- Primary：清晰的 Material 蓝，用于选中态、主操作、焦点和关键数字。
- Secondary：青绿，用于本地、只读和可用状态。
- Background：柔和中性浅灰蓝，降低长时间使用疲劳。
- Surface：白色或近白色，用于导航、内容区、卡片和详情区。
- Text Primary：深灰黑，不使用纯黑压满页面。
- Text Secondary：中灰，用于说明、来源和次级状态。
- Success / Warning / Error：分别表示正常、需要注意、需要处理。

## 页面结构

- 页面为 100dvh 桌面应用，不允许 body 或整页滚动。
- 左侧导航、中心工作区和右侧详情区是主要表面。
- 卡片使用软边界、轻阴影或低对比描边，避免硬线框堆叠。
- 常规容器圆角 20-28px；小控件圆角可更紧凑。
- 不使用随机渐变、玻璃拟态堆叠或强装饰背景。

## 间距

- 使用 8px 间距网格。
- 控制台外边距 12-16px。
- 面板内部间距 12-20px。
- 卡片内边距 14-18px。
- 信息组之间留出明显间隔，避免表格行密度。

## 主导航

主导航只包含：

- 首页。
- 技能。
- MCP。
- 高级。

导航项必须显示中文名称。数量只显示普通用户能理解的计数，例如技能数量、MCP 服务数量、MCP 工具数量和需要处理数量。

## 首页

首页必须突出：

- AI 技能数量。
- MCP 服务数量。
- MCP 工具数量。
- 需要处理的项目数量。
- 一键查找入口。
- 简短隐私说明。

首页不得展示原始诊断、内部日志或历史示例入口。

## 技能卡片

技能卡片必须显示：

- 技能名称。
- 一句话用途。
- 来源。
- 哪个 AI 工具可以使用它。
- 使用入口。
- 状态。

高级来源信息可以折叠显示，但不能压过普通用户的用途说明。

## MCP 卡片

MCP 卡片必须显示：

- 服务名称。
- 工具数量。
- 配置来源。
- 简短用途。
- 状态。
- 需要处理的问题。

页面不得启动或连接 MCP 服务，不得显示环境变量值或密钥。

## 高级页

高级页可以展示：

- 来源分组。
- 手动添加查找位置。
- 跳过项摘要。
- 错误摘要。
- 本地数据控制。
- 开发者补充信息。

这些内容不能出现在普通用户的默认首页。

## 搜索

- 顶部产品头部包含搜索栏。
- 输入提示文案使用中文。
- 搜索范围包括名称、用途、来源、状态和使用入口。
- 搜索是筛选，不执行命令。

## 状态标签

- 可用：可以使用。
- 需要处理：需要用户查看。
- 重复：发现多个相同或近似项目。
- 已损坏：来源缺文件、格式异常或无法读取。
- 来源不明：还不能判断来自哪里。

## 动效

- 使用轻量 Material motion：页面进入、模块切换、详情切换、复制反馈。
- 动效只使用透明度与 transform。
- 必须尊重 `prefers-reduced-motion`。
- 禁止滚动动画与 ScrollTrigger。

## 禁止项

- 密集后台表格作为主界面。
- 随机渐变和装饰性背景。
- 不一致卡片尺寸。
- 面向用户的英文 UI 标签。
- 整页滚动。
- 把脚本、报告、策略、检查器、旧示例或原始诊断作为普通用户主导航。
- 全局写入。
- 恢复或重建 Codex 自动化。
- 修改全局技能入口、MCP、认证、供应商、模型、launchd、浏览器扩展、凭据或环境配置。
